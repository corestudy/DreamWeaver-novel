# 接口变更检查清单

用途：任何涉及 `/api/*` 的改动，在提交前必须完成以下检查。

## 必做项
- [ ] 后端路由已实现且可访问（`server.js`）。
- [ ] 前端调用已对齐 method/path（`public/*.js`）。
- [ ] 已更新 `前后端交互标准文档.md`：
  - [ ] 核心接口清单包含该接口。
  - [ ] 前端接口契约矩阵包含该接口（method/path/request/response/error）。
  - [ ] 文档版本号和日期已更新。
  - [ ] 最近变更摘要已记录本次改动。
- [ ] 本地执行 `npm.cmd run check` 通过（含 `check:api-doc`）。

## 发布前建议
- [ ] 关键路径手测：书架、默认模式、开挂模式、outline、workspace、导出。
- [ ] 若接口有兼容性变更，补充迁移说明（旧接口是否保留、何时废弃）。

## 失败处理
- `check:api-doc` 失败时，优先修复文档缺项，不要跳过检查直接发布。
