# 后续更新建议（2026-03-15）

本建议面向当前 `novel-writing-assistant-demo` 的下一阶段迭代，目标是：稳定可用、上下文连贯、可持续打包发布。

## P0（先做，影响可用性）

### 1) 彻底解决 `better-sqlite3` 与 Node 版本不匹配
- 建议：
  - 锁定开发与打包使用的 Node 主版本（建议 LTS，统一到 `.nvmrc` 或文档强约束）。
  - 在 `postinstall` 增加 `electron-builder install-app-deps` 或等效重建流程。
  - 打包前固定执行一次原生模块重建检查（脚本化）。
- 验收标准：
  - `npm.cmd run dev` 不再出现 `NODE_MODULE_VERSION` 错误。
  - 新机器拉取项目后，一次安装即可启动。

### 2) API 路由可用性守护（避免 “route not found”）
- 建议：
  - 增加后端启动时路由自检日志（输出关键路由注册清单）。
  - 前端统一处理 404 API 错误，提示“需重启服务”并可一键重试。
  - 增加最小 API 健康检查端点（如 `/api/health`）。
- 验收标准：
  - 关键流程（书籍创建、大纲、工作台、cheat-mode）启动后都能一次访问成功。

### 3) 文档与实现强一致
- 建议：
  - 设定“每次改接口必改文档”的检查项（PR 模板或发布清单）。
  - `前后端交互标准文档.md` 增加“最近变更摘要 + 版本号 + 日期”。
- 验收标准：
  - 任意一个前端接口都可在文档中查到 method/path/request/response/error。

## P1（随后做，提升核心体验）

### 4) 每本书上下文记忆管理可观测化
- 建议：
  - 在工作台显示“上下文注入状态”（成功/失败/最近注入时间）。
  - 提供“重新发送上下文”按钮，并记录注入日志（仅本地）。
  - 注入来源固定为：大纲 + 设定资料 + 最近章节摘要。
- 验收标准：
  - 用户可明确知道本次 AI 生成是否使用了当前书籍上下文。

### 5) AI 写作统一入口完善
- 建议：
  - 保持“AI写作”单入口，但明确动作类型（写作/改写/续写/润色/总结）。
  - “写作”动作默认把结果自动插入编辑器，并提供撤销提示。
  - 对空指令、超长指令、API 超时做前端可读错误提示。
- 验收标准：
  - 用户点击执行后，结果可预测、有反馈、可恢复。

### 6) 大纲与设定同步机制固化
- 建议：
  - 统一“设定资料”的单一数据源，outline 与 workspace 都读取同一存储。
  - 保存后立即广播刷新（或轮询最小化实现）。
- 验收标准：
  - 两个页面编辑同一本书设定时，不再出现数据不同步。

## P2（增强项，发布前做）

### 7) 导出能力增强
- 建议：
  - 扩展导出格式（现有 txt/md/html 基础上增加 docx，若成本可控）。
  - 增加导出模板（章节标题、分隔符、元数据开关）。
- 验收标准：
  - 用户可直接导出投稿/备份可用的文件结构。

### 8) 隐私与轻量化继续收敛
- 建议：
  - 保持默认不提交 `storage/`、用户 API 配置、本地缓存。
  - 打包前自动执行 `npm.cmd run clean:user-data` 与 `npm.cmd run lightweight`。
  - 增加“隐私检查脚本”（扫描 token/key 常见模式）。
- 验收标准：
  - 发布包不含个人配置、密钥、历史聊天隐私数据。

### 9) 最小自动化测试
- 建议：
  - 增加 3 类基础测试：API 冒烟、关键页面脚本加载、导出功能。
  - CI 至少运行 `npm.cmd run check` + 冒烟测试。
- 验收标准：
  - 关键回归（按钮失效、路由缺失、导出失败）可在发布前被拦截。

## 推荐执行顺序（两周版）

1. 第 1-3 天：P0-1、P0-2（先保启动和路由稳定）。
2. 第 4-5 天：P0-3 + 文档校对。
3. 第 6-9 天：P1-4、P1-5（记忆状态与 AI 写作体验）。
4. 第 10-12 天：P1-6 + 回归修复。
5. 第 13-14 天：P2-8、P2-9，准备发布。

## 变更管理建议

- 每完成一个建议项，同时更新：
  - `前后端交互标准文档.md`（接口变更）
  - `README.md`（用户可见功能变更）
  - `常用命令.md`（新增脚本/流程）
- 发布前固定执行：
  - `npm.cmd run check`
  - `npm.cmd run dev` 全流程手测
  - `npm.cmd run build:desktop`
