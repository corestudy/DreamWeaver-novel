## 2. 通用规则
- 请求头: `Content-Type: application/json`
- 请求体上限: `12mb`
- 成功状态码: `200 / 201`
- 失败状态码: `400 / 404 / 500`
- CORS: 仅允许 `http://localhost:3000` 与 `http://127.0.0.1:3000`（以及无 Origin 的本地调用）
- 统一错误结构:

```json
{
  "error": "error message"
}
```

## 5. 大纲与上传文本接口

### 5.1 POST `/api/projects/:projectId/outline/from-text`
用途：从上传小说文本生成大纲。

请求:
```json
{
  "novelText": "小说正文文本（必填）",
  "promptHint": "可选"
}
```

响应:
```json
{
  "outline": "# 整理后的大纲...",
  "source": "api:gpt-4o-mini"
}
```

### 5.2 POST `/api/projects/:projectId/outline/from-text/context`（新增）
用途：从上传小说文本提取人物/地点/规则，并覆盖当前书籍设定资料。

请求:
```json
{
  "novelText": "小说正文文本（必填）",
  "promptHint": "可选"
}
```

响应:
```json
{
  "ok": true,
  "source": "api:gpt-4o-mini",
  "characters": [{ "id": 1, "name": "林澈", "profile": "..." }],
  "locations": [{ "id": 1, "name": "旧港", "description": "..." }],
  "rules": [{ "id": 1, "key_name": "夜行规则", "value_text": "..." }]
}
```

## 7. 本书独立聊天（v3.0 新增）
- 聊天数据按 `project_id` 存储在 `project_chat_messages`，关闭软件后仍保留。
- 聊天接口会自动注入当前书籍上下文（来源对齐 `outline.html` 设定）：
  - 本书大纲
  - 本书设定资料（人物/地点/规则）
  - 本书历史聊天消息

### 7.1 GET `/api/projects/:projectId/chat/messages`
用途：读取当前书籍聊天记录（按时间正序返回）。

响应示例:
```json
[
  {
    "id": 1,
    "project_id": 3,
    "role": "user",
    "content": "请帮我规划第一卷冲突",
    "source": "user",
    "created_at": "2026-03-14 08:00:00"
  }
]
```

### 7.2 POST `/api/projects/:projectId/chat/messages`
用途：发送一条用户消息，并返回 AI 回复与最新消息列表。

请求:
```json
{
  "message": "请基于当前大纲给我第3章转折方案"
}
```

响应:
```json
{
  "ok": true,
  "source": "api:gpt-4o-mini",
  "userMessage": { "id": 10, "role": "user", "content": "..." },
  "assistantMessage": { "id": 11, "role": "assistant", "content": "..." },
  "messages": []
}
```

### 7.3 DELETE `/api/projects/:projectId/chat/messages`
用途：清空当前书籍聊天记录。

响应:
```json
{
  "ok": true,
  "projectId": 3
}
```

### 7.4 POST `/api/projects/:projectId/ai/compose`（新增，工作台主入口）
用途：工作台「AI写作」主接口；将“本书聊天指令 + 写作生成”合并为一个接口，生成时固定注入本书 `outline` 上下文（大纲+设定资料）。

请求:
```json
{
  "chapterId": 12,
  "action": "continue",
  "selectedText": "可选，编辑器选中文本",
  "instruction": "可选，本书聊天/创作指令",
  "tone": "可选，语气",
  "targetLength": "可选，长度要求"
}
```

响应:
```json
{
  "output": "AI 生成正文",
  "source": "api:gpt-4o-mini",
  "messages": []
}
```

说明:
- `instruction` 会作为本书聊天消息写入历史，并参与本次写作生成提示。
- 生成结果会同时写入本书聊天历史与生成日志。
- `POST /api/ai/action` 仍保留兼容旧调用，但工作台默认改为 `ai/compose`。

## 8. 记忆管理（v3.4 新增）
目的：确保 AI 接口已读取当前书籍上下文（大纲 + 设定资料 + 连贯性素材）。

### 8.1 GET `/api/projects/:projectId/memory/status`
用途：读取本书记忆校验状态。

响应:
```json
{
  "status": "ok",
  "upToDate": true,
  "contextHash": "8a1f...f2",
  "checkedAt": "2026-03-14 21:30:00",
  "syncedAt": "2026-03-14T21:30:00.000Z",
  "source": "api:gpt-4o-mini",
  "detail": "AI 已返回正确上下文回执",
  "snapshot": {
    "hasOutline": true,
    "characterCount": 6,
    "locationCount": 4,
    "ruleCount": 8,
    "summaryCount": 3,
    "factCount": 12
  }
}
```

### 8.2 POST `/api/projects/:projectId/memory/check`
用途：触发一次实时校验，请 AI 回传 `nonce + context_hash` 回执，判断是否已读取当前上下文。

请求:
```json
{}
```

响应:
```json
{
  "ok": true,
  "status": "ok",
  "upToDate": true,
  "contextHash": "8a1f...f2",
  "detail": "AI 已返回正确上下文回执",
  "source": "api:gpt-4o-mini",
  "checkedAt": "2026-03-14 21:30:00",
  "syncedAt": "2026-03-14T21:30:00.000Z",
  "snapshot": {},
  "receipt": {
    "ack": "7f2d8f...",
    "context_hash": "8a1f...f2",
    "outline_present": true,
    "character_count": 6,
    "location_count": 4,
    "rule_count": 8
  }
}
```

### 8.3 POST `/api/projects/:projectId/memory/resend`
用途：重新发送当前上下文到 AI，并立即执行同样的回执校验。

请求:
```json
{}
```

响应:
```json
{
  "ok": false,
  "status": "failed",
  "upToDate": false,
  "detail": "未配置可用 API Key，无法验证上下文读取",
  "action": "resend"
}
```

## 10. 导出能力说明（无新增 HTTP API）
- 导出不走后端 HTTP 接口，不写入服务端数据库。
- 桌面版通道：
  - Renderer 调用：`window.novelDesktop.saveExportFile(...)`
  - Electron IPC: `export:saveFile`
  - Main 进程弹窗选择路径并写入文件
- 兼容性：
  - 在非 Electron（纯浏览器）中，自动使用 Blob 下载。

## 12. 设定资料同步说明（outline 与 workspace）
- 两个页面都使用同一套接口：`/api/projects/:projectId/context`。
